# Use Cases

FDGF is a reusable technical framework, not a single reporting implementation. The same primitives can support multiple regulated-data workflows across U.S. financial institutions.

## Regional Bank Reporting Workflow

- validate loan, deposit, and exposure extracts before scheduled regulatory reporting
- track how source-system data was transformed into reportable aggregates
- generate machine-readable evidence for reviewers, risk teams, and internal audit

## Large-Bank Basel III Workflow

- apply contract-driven checks to exposure and risk-weight data
- preserve transformation lineage for daily or monthly calculation runs
- summarize critical failures before downstream reporting and review steps

## CCAR / Stress Testing Data Controls

- validate scenario and model-input data
- record workflow lineage for scenario rollups
- summarize outputs used in model-governance and control review

## Credit-Risk Model Monitoring

- calculate drift metrics on scored populations
- run simple fairness checks on approval or segmentation outputs
- publish concise monitoring summaries for model-governance review

## Machine-Readable Reporting Preparation

- standardize field-level validation and formatting rules
- create interoperable JSON evidence artifacts
- reuse the same control model across multiple datasets, reporting workflows, and teams
